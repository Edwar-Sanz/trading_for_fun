Archivos .mqh vs Archivos .mq5

Existen dos tipos principales de archivos de código fuente: .mqh y .mq5. A continuación, se explica la diferencia entre ellos y su uso.

Archivos .mq5

Los archivos con extensión .mq5 contienen la implementación completa de un programa ejecutable, ya sea un Asesor Experto (Expert Advisor), un Indicador o un Script, estos se compilan directamente en archivos .ex5, que son los ejecutables utilizados en MetaTrader 5. En los archivos .mq5 se importan los importar archivos .mqh para reutilizar código y mejorar la modularidad.

Archivos .mqh

Los archivos con extensión .mqh son archivos de cabecera o bibliotecas. Su propósito principal es permitir la reutilización de código mediante la encapsulación de funciones, clases, constantes y macros que pueden ser utilizadas en múltiples archivos .mq5.

Ejemplo

En el archivo .mqh:


    // Archivo: MyLibrary.mqh

    void PrintHello()
    {
        Print("¡Hola mundo!");
    }

Incluimos el archivo .mqh en el archivo .mq5:


    // Archivo: MyExpert.mq5
    #include <MyLibrary.mqh>

    void OnStart()
    {   
        // función definida en MyLibrary.mqh
        PrintHello(); 
    }

Conclusión

  • .mq5: Contienen la lógica principal del programa y se compilan directamente en ejecutables .ex5.

  • .mqh: Son archivos de cabecera que permiten la reutilización de código en múltiples archivos .mq5.
